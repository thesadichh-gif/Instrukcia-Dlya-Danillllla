# Instrukcia-Dlya-Danillllla: Подробная пошаговая инструкция по выполнению Лабораторной работы №3 «Протоколы ARP и ICMP»

---

## ВАЖНО: Что нам понадобится

Из Лабораторной работы №2 у вас уже должны быть установлены:
- Oracle VirtualBox с GNS3 VM
- GNS3 GUI
- Wireshark

Если ничего не установлено — сначала выполните установку по инструкции к ЛР №2.

---

## ЧАСТЬ 1: Запуск окружения

### Шаг 1: Запуск GNS3 VM

1. Откройте **Oracle VirtualBox**
2. Найдите в списке **GNS3 VM** и нажмите **Запустить**
3. Дождитесь полной загрузки — зелёный экран с IP-адресом
4. **Не закрывайте** окно VM — оставьте работать в фоне

---

### Шаг 2: Запуск GNS3

1. Запустите **GNS3** с рабочего стола
2. В панели **Servers Summary** (правый нижний угол) убедитесь — оба кружка **зелёные** 🟢
3. Если кружки красные — подождите 30–60 секунд

---

## ЧАСТЬ 2: Построение топологии сети

### Итоговая топология которую нужно построить:

```
PC1 ─┐
PC2 ─┤
PC3 ─┼── Switch1 ── R1 (маршрутизатор FRR) ── Switch2 ──┬── PC6
PC4 ─┤                                                    ├── PC7
PC5 ─┘                                                    └── PC8
```

**Левая подсеть:** 192.168.3.0/24 (PC1–PC5 + интерфейс R1)
**Правая подсеть:** 192.168.5.0/24 (PC6–PC8 + интерфейс R1)

---

### Шаг 3: Создание нового проекта

1. В GNS3 нажмите **File → New blank project**
2. В поле **Name** введите: `Lab3_ARP_ICMP`
3. Нажмите **OK**

---

### Шаг 4: Добавление 8 компьютеров VPCS

1. На левой панели нажмите иконку **End Devices** (монитор)
2. Найдите **VPCS** в списке
3. Перетащите (или кликните и кликните на рабочей области) **8 раз** — появятся PC1–PC8
4. Расположите их примерно так:

```
Левая часть экрана:     Правая часть экрана:
PC1                          PC6
PC2                          PC7
PC3                          PC8
PC4
PC5
```

> 💡 Устройства можно перетаскивать мышью в любое место

---

### Шаг 5: Добавление двух коммутаторов

1. На левой панели нажмите иконку **Switches**
2. Найдите **Ethernet switch**
3. Перетащите его в **центр-левее** рабочей области → появится **Switch1**
4. Ещё раз перетащите **Ethernet switch** в **центр-правее** → появится **Switch2**

---

### Шаг 6: Добавление маршрутизатора FRR

1. На левой панели нажмите иконку **Routers** (круговые стрелки)
2. Найдите **FRR 8.2.2** в списке
3. Перетащите его **между Switch1 и Switch2** на рабочую область
4. В диалоге Server выберите **GNS3 VM** → нажмите **OK**
5. Появится маршрутизатор **FRR8.2.2-1** (или **R1**)

> ⚠️ Если FRR нет в списке — вернитесь к инструкции по установке FRR из предыдущего раздела

## ЧАСТЬ 6: Проверка №1 — ping внутри одной подсети

### Шаг 26: Ping с PC1 (192.168.3.3) на PC3 (192.168.3.5)

1. Перейдите в консоль **PC1** (там написано `192.168.3.3>`)
2. Введите команду и нажмите **Enter**:
```
ping 192.168.3.5
```
3. Дождитесь выполнения (5 пакетов)
4. Ожидаемый результат:
```
84 bytes from 192.168.3.5 icmp_seq=1 ttl=64 time=0.143 ms
84 bytes from 192.168.3.5 icmp_seq=2 ttl=64 time=0.270 ms
84 bytes from 192.168.3.5 icmp_seq=3 ttl=64 time=0.186 ms
84 bytes from 192.168.3.5 icmp_seq=4 ttl=64 time=0.185 ms
84 bytes from 192.168.3.5 icmp_seq=5 ttl=64 time=0.189 ms
```

---

### Шаг 27: Изучение пакетов в Wireshark (ping внутри сети)

В окне Wireshark вы увидите пакеты. Изучите их:

**Первый пакет — ARP запрос:**
1. Найдите строку с **ARP** и столбцом Destination = **Broadcast**
2. Кликните на неё
3. В средней панели разверните **Address Resolution Protocol (request)**
4. Изучите поля:
   - `Sender IP address: 192.168.3.3` — кто спрашивает
   - `Target IP address: 192.168.3.5` — кого ищет
   - `Target MAC address: 00:00:00:00:00:00` — MAC ещё неизвестен
   - `Opcode: request (1)` — это запрос

**Второй пакет — ARP ответ:**
1. Найдите строку **ARP** где Source = `192.168.3.5`
2. Кликните на неё
3. Разверните **Address Resolution Protocol (reply)**
4. Изучите:
   - `Sender IP address: 192.168.3.5` — кто отвечает
   - `Sender MAC address: xx:xx:xx:xx:xx:xx` — вот его MAC!
   - `Opcode: reply (2)` — это ответ

**Пакеты ICMP:**
1. Найдите строку **ICMP** с Info = `Echo (ping) request`
2. Кликните на неё
3. Разверните **Internet Protocol Version 4**:
   - `Source Address: 192.168.3.3`
   - `Destination Address: 192.168.3.5`
   - `Time to Live: 64`
4. Разверните **Internet Control Message Protocol**:
   - `Type: 8 (Echo ping request)` — тип 8 = запрос
   - `Code: 0`
   - `Sequence Number` — номер пакета

5. Найдите строку **ICMP** с Info = `Echo (ping) reply`
6. Разверните **Internet Control Message Protocol**:
   - `Type: 0 (Echo ping reply)` — тип 0 = ответ

---

## ЧАСТЬ 7: Проверка №2 — ping во внешнюю сеть (через маршрутизатор)

### Шаг 29: Запуск нового захвата Wireshark на PC2

1. В **Topology Summary** найдите **192.168.3.4** (PC2)
2. Раскройте его соединения
3. **Правой кнопкой** на соединение с Switch1 → **Start capture** → OK
4. В новом окне Wireshark установите фильтр: `arp || icmp`

---

### Шаг 30: Ping с PC2 (192.168.3.4) на PC8 (192.168.5.5)

1. Перейдите в консоль **PC2** (`192.168.3.4>`)
2. Введите:
```
ping 192.168.5.5
```
3. Нажмите **Enter** и дождитесь результата
4. Первый пакет может дать `timeout` — это нормально (маршрутизатор тоже делает ARP):
```
192.168.5.5 icmp_seq=1 timeout
84 bytes from 192.168.5.5 icmp_seq=2 ttl=63 time=12.109 ms
84 bytes from 192.168.5.5 icmp_seq=3 ttl=63 time=16.535 ms
84 bytes from 192.168.5.5 icmp_seq=4 ttl=63 time=19.407 ms
84 bytes from 192.168.5.5 icmp_seq=5 ttl=63 time=15.087 ms
```

> 💡 Обратите внимание: TTL теперь **63** (а не 64). Это потому что пакет прошёл через маршрутизатор и TTL уменьшился на 1.

---

### Шаг 31: Изучение пакетов межсетевого ping в Wireshark

В Wireshark (захват с PC2) вы увидите:

**ARP запрос к маршрутизатору:**
- Source = MAC адрес PC2
- Destination = **Broadcast**
- В деталях: `Target IP address: 192.168.3.1` — PC2 ищет MAC маршрутизатора, а не PC8!
- Это потому что PC8 в другой сети, и пакет нужно сначала отправить на шлюз

**ARP ответ от маршрутизатора:**
- Source = MAC адрес R1 (eth0)
- Теперь PC2 знает MAC маршрутизатора

**ICMP запрос:**
- Source: `192.168.3.4`
- Destination: `192.168.5.5`
- Destination MAC: MAC маршрутизатора (не PC8!)
- Type: 8 (запрос)

---

### Шаг 32: Трассировка маршрута командой trace

В консоли **PC3** (`192.168.3.5>`) введите:
```
trace 192.168.5.4
```
Нажмите **Enter** и дождитесь результата:
```
trace to 192.168.5.4, 8 hops max, press Ctrl+C to stop
1  *192.168.3.1  3.753 ms  9.463 ms
2  **192.168.5.4  15.613 ms (ICMP type:3, code:3, Destination port unreachable)
```

> 💡 **Объяснение результата:**
> - Строка 1: первый хоп — маршрутизатор R1 (192.168.3.1)
> - Строка 2: второй хоп — конечный хост PC7 (192.168.5.4)
> - Звёздочки означают что некоторые пакеты не вернулись в срок
> - `ICMP type:3, code:3` — это сообщение «порт недостижим» от конечного хоста, которое trace использует как сигнал что назначение достигнуто

## ЧАСТЬ 8: Проверка №3 — ping на несуществующий хост в существующей сети

### Шаг 33: Запуск захвата с PC5 (192.168.3.7)

1. В Topology Summary найдите **192.168.3.7** (PC5)
2. **Правой кнопкой** → соединение со Switch1 → **Start capture** → OK
3. Установите фильтр Wireshark: `arp || icmp`

---

### Шаг 34: Ping на несуществующий хост 192.168.5.6

В консоли **PC5** (`192.168.3.7>`) введите:
```
ping 192.168.5.6
```
Нажмите **Enter**. Ожидаемый результат — таймаут:
```
192.168.5.6 icmp_seq=1 timeout
192.168.5.6 icmp_seq=2 timeout
192.168.5.6 icmp_seq=3 timeout
192.168.5.6 icmp_seq=4 timeout
192.168.5.6 icmp_seq=5 timeout
```

> 💡 **Почему timeout?** Адрес 192.168.5.6 не существует. Маршрутизатор получает пакет, пытается найти MAC-адрес 192.168.5.6 через ARP, но никто не отвечает. Пакет теряется. Маршрутизатор в данном случае не отправляет ICMP «хост недостижим».

---

### Шаг 35: Ping на хост в несуществующей сети

В консоли **PC4** (`192.168.3.6>`) введите:
```
ping 192.168.6.6
```
Нажмите **Enter**. Ожидаемый результат — «хост недостижим»:
```
*192.168.3.1 icmp_seq=1 ttl=255 time=9.803 ms (ICMP type:3, code:1, Destination host unreachable)
*192.168.3.1 icmp_seq=2 ttl=255 time=5.410 ms (ICMP type:3, code:1, Destination host unreachable)
*192.168.3.1 icmp_seq=3 ttl=255 time=6.367 ms (ICMP type:3, code:1, Destination host unreachable)
*192.168.3.1 icmp_seq=4 ttl=255 time=6.153 ms (ICMP type:3, code:1, Destination host unreachable)
*192.168.3.1 icmp_seq=5 ttl=255 time=6.361 ms (ICMP type:3, code:1, Destination host unreachable)
```

> 💡 **Почему «хост недостижим»?** Сеть 192.168.6.0 вообще не настроена на маршрутизаторе. Маршрутизатор не знает куда направить пакет и сразу отвечает ICMP сообщением типа 3, код 1 = «Destination host unreachable».

---

### Шаг 36: Изучение пакета ICMP «хост недостижим» в Wireshark

В Wireshark (захват с PC4) найдите строку ICMP с Info = `Destination unreachable`

Кликните на неё и изучите в средней панели:

```
Internet Protocol Version 4:
  Source Address: 192.168.3.1    ← маршрутизатор отправил этот пакет
  Destination Address: 192.168.3.6  ← обратно на PC4

Internet Control Message Protocol:
  Type: 3 (Destination unreachable)  ← тип 3
  Code: 1 (Host unreachable)         ← код 1 = хост недостижим
```

## ТИПИЧНЫЕ ПРОБЛЕМЫ

| Проблема | Причина | Решение |
|---------|---------|---------|
| Ping между PC и R1 не работает | Маршрутизатор не настроен | Повторите настройку eth0/eth1 |
| Ping во внешнюю сеть не работает | Не указан шлюз на PC | Добавьте шлюз в команде ip: `ip адрес маска шлюз` |
| FRR консоль не открывается | VM ещё грузится | Подождите 2–3 минуты |
| Команда `ip address` не работает в FRR | Неверный синтаксис | Используйте `/24` вместо `255.255.255.0` |
| Wireshark пустой | Не запущены устройства | Нажмите ▶ Run |
| trace показывает только звёздочки | Маршрутизатор блокирует | Нормально, подождите — результат всё равно появится |
